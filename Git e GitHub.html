Git e GitHub Git= Sistema de versionamento de código GitHub= Site de Hospedágem de código Branch= São linhs do tempo do código trabalhado, tendo a principal ou as paralelas para testar o código sem afetar a linha principal. Commit= Os commits são as alteraçoes que são feitas no código para ter um controle e salvar as alterações. Merge= Merge é quando fazemos uma junção entre as Brunchs ou de uma Branch para outra. Remote= É quando faz uma ponte entre o Git e o GitHub (ou outro site de hospedágem). Push= Manda as alterações do código para o GitHub. Pull= Faz ao contrário, puxa essas alterações para a máquina. ## Alguns Comandos git init= Cria um repositório / incia o projeto. git add "nome do arquivo"= Adiciona um arquivo na Branch. git add .= Adiciona todos os arquivos da pasta. git status= Mostra o status do ultimo comando. git remote add orgin (link do GitHub)= Faz a conexão do Git com o GitHub. git pull= Puxa as alterações do site para a máquina. git push= Envia a ultima alteração para o site. git push -u origin main ou master= Com git push origin main ou master você diz ao git para enviar todos os commits no branch local atualmente verificado para o repositório remoto identificado pelo nome origin em seu branch remoto nomeado main ou master. git commit -m= Cria um novo commit e dá um nome para ele. git branch (nome da branch)= Cria uma nova branch. gt branch -M= Altera o nome da Branch. git checkout (nome da branch)= Troca de Branch. git checkout -b "Nome da Branch"= Sai da Branch atual, cria e entra em uma nova Branch cujo nome foi escolhido. git clone= clona um código para sua máquina. git commit --amend= Edita a mensagem do commit. git log= Mostra as versões(Commits) do código. git reset --hard (hash do commit)= Volta para o Commit desejado.
