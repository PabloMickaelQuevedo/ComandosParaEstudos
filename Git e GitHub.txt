Git e GitHub

Git= Sistema de versionamento de código
GitHub= Site de Hospedágem de código
Branch= São linhs do tempo do código trabalhado, tendo a principal ou as paralelas para testar o código sem afetar a linha principal.
Commit= Os commits são as alteraçoes que são feitas no código para ter um controle e salvar as alterações.
Merge= Merge é quando fazemos uma junção entre as Brunchs ou de uma Branch para outra.
Remote= É quando faz uma ponte entre o Git e o GitHub (ou outro site de hospedágem).
Push= Manda as alterações do código para o GitHub.
Pull= Faz ao contrário, puxa essas alterações para a máquina. 

Push do GIT no terminal: 

Para usar o push do Git no terminal do linux, devemos cadastrar uma chave SSH.
Primeiro devemos entrar no nosso GitHub, entrar em configurações, SSH and GPG keys e criar uma chave SSH.
Agora devemos ir no terminal e criar uma chave SSH na máquina. Para isso usamos os comandos abaixo.
ssh-keygen = Vai gerar a chave SSH (detalhe, vai gerar a chave no local que estiver o terminal), agora colocamos um nome para essa chave e (opcional)uma senha.
Vão ser gerados 2 arquivos com o nome que foi escolhido, um deles vai ser .pub, esse .pub vamos usar para colocar na nossa SSH que criamos no GitHub.
cat nome-do-arquivo.pub = vai aparecer os caracteres da chave, então copia e cola onde pede a chave la no GitHub.
eval 'ssh-agent' = Para colocar dentro as chaves
ssh-add nome-do-arquivo = Adiciona a chave privada no caso sem o .pub.
Podemos criar a chave la no GitHub e seguir adiante.
Agora é importante pegar o link do SSH no GitHub ao dar o remote, e não o HTML.
Fica assim >> git remote add origin "link do SSH not GitHub sem aspas".
Pronto!!! agora podemos dar um push.

Alguns Comandos

git init= Cria um repositório / incia o projeto.
git add "nome do arquivo"= Adiciona um arquivo na Branch.
git add .= Adiciona todos os arquivos da pasta.
git status= Mostra o status do ultimo comando.
git remote add orgin (link do GitHub)= Faz a conexão do Git com o GitHub.
git pull= Puxa as alterações do site para a máquina.
git push= Envia a ultima alteração para o site.
git push -u origin main ou master= Com git push origin main ou master você diz ao git para enviar todos os commits no branch local atualmente verificado para o repositório remoto identificado pelo nome origin em seu branch remoto nomeado main ou master.
git commit -m= Cria um novo commit e dá um nome para ele.
git branch (nome da branch)= Cria uma nova branch.
gt branch -M= Altera o nome da Branch. 
git checkout (nome da branch)= Troca de Branch.
git checkout -b "Nome da Branch"= Sai da Branch atual, cria e entra em uma nova Branch cujo nome foi escolhido.
git clone= clona um código para sua máquina.
git commit --amend= Edita a mensagem do commit.
git log= Mostra as versões(Commits) do código.
git reflog= Mostra as Hash dos commits para poder navegar pelas versões.
git reset --hard (hash do commit)= Volta para o Commit desejado.
